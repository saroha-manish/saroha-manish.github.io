C5i Project - Inventory Management Optimization

Project Overview

The C5i project focuses on optimizing inventory management and preventing stockouts by leveraging predictive modeling and time-series forecasting techniques. The goal was to build a system capable of forecasting stock levels based on historical data, external factors, and real-time data streams. The solution improved inventory accuracy by 30%, reducing stockouts by 25% and enhancing overall supply chain operations.

Key Features

  • Time-Series Forecasting: Applied advanced time-series forecasting models to predict stock levels based on historical inventory data, demand patterns, and weather conditions.
  • Real-Time Data Integration: Integrated external data streams (weather, promotions, and demand forecasts) to adjust predictions in real-time, ensuring more accurate forecasting.
  • Improved Accuracy: Achieved 30% improvement in stock forecast accuracy, helping to optimize inventory management and reduce stockouts by 25%.
  • Dynamic Adjustments: The system dynamically adjusts inventory predictions based on real-time data, ensuring up-to-date stock levels.

Technologies and Tools

  • Programming Languages: Python
  • Machine Learning Libraries: TensorFlow, Scikit-learn
  • Data Handling: Pandas, NumPy
  • Databases: SQL for data storage and retrieval
  • Containers: Docker for system deployment and scalability
  • Development Tools: Jupyter Notebooks for exploratory data analysis and model development

Problem Statement

Traditional inventory management systems often fail to predict stock levels accurately, leading to frequent stockouts. This disrupts supply chains, increases operational costs, and negatively impacts customer satisfaction. The C5i project aimed to solve this problem by developing a predictive model to optimize inventory management.

Solution Approach

Collected and preprocessed historical inventory data, sales data, and external factors like weather and promotions. Built a time-series forecasting model using TensorFlow to predict future stock levels. Designed a system to integrate real-time data from various sources (weather, promotions, and demand forecasts) to continuously adjust predictions. The system was deployed using Docker for easy scalability and integration with existing inventory management systems.

Results

  • 30% improvement in forecasting accuracy, leading to better stock level predictions.
  • 25% reduction in stockouts, which increased operational efficiency and reduced inventory costs.
  • Real-time adjustments allowed for better decision-making, reducing overstocking and stockouts.

Challenges

Handling missing or noisy data required the implementation of advanced data preprocessing techniques. Integrating multiple data streams in real time required robust pipeline architecture, solved using Docker for deployment.

Future Work

Expand the model to include real-time inventory tracking for even more accurate predictions. Integrate with ERP and supply chain management systems for seamless automation and decision-making.

Landis+Gyr Project - QoE Monitoring with MOS Estimation

Project Overview

This project at Landis+Gyr aimed to enhance network service quality by improving visibility into user experience and proactively detecting network issues. A machine learning-based system was developed to estimate Mean Opinion Score (MOS), allowing real-time monitoring of Quality of Experience (QoE) and enabling early detection of service degradation.

Key Features

  • MOS Prediction: Developed a predictive model achieving 94% accuracy in estimating MOS for network sessions.
  • Model Optimization: Tuned and compared multiple regression algorithms including Random Forest, XGBoost, and LSTM for optimal performance.
  • Real-Time Dashboards: Integrated predictions into operational dashboards to monitor QoE in real time and trigger alerts for degradations.

Technologies and Tools

  • Programming Languages: Python
  • Machine Learning Libraries: Scikit-learn, XGBoost, TensorFlow
  • Data Processing: Pandas, NumPy
  • Visualization: Plotly, Dash
  • Deployment: Docker for scalable service deployment
  • Monitoring Tools: Grafana for dashboard integration and alerting

Problem Statement

Lack of real-time visibility into user experience and delays in detecting network issues led to reduced service quality and increased customer complaints. Traditional monitoring tools failed to capture subjective QoE indicators like MOS, limiting proactive response capabilities.

Solution Approach

Collected historical network performance metrics and user feedback to train supervised learning models capable of estimating MOS. Multiple models were evaluated and optimized for accuracy and robustness, and the final solution was integrated into dashboards with real-time monitoring capabilities. Alerts were configured to flag degradation proactively.

Results

  • 94% accuracy in MOS estimation, enabling reliable QoE tracking.
  • 22% reduction in customer complaints due to early issue detection.
  • Improved SLA compliance and increased user satisfaction through proactive network management.

Challenges

Dealing with subjective QoE labels required data cleaning and balancing techniques. Integrating predictions with existing dashboards and ensuring low-latency model inference for real-time use posed architectural challenges, which were solved using Docker-based deployment and efficient API integration.

Future Work

Plan to incorporate real-time user feedback and expand the model to support anomaly detection for broader network health monitoring. Explore self-healing mechanisms based on MOS thresholds and automated issue resolution workflows.
