LandisGyr Project - MOS prediction

Project Overview

The project focused on developing a predictive system to estimate the Mean Opinion Score (MOS) for proactive network management. By optimizing machine learning models, it reduced customer complaints by 22% and improved overall service quality.

Key Features

  • MOS Prediction: Developed machine learning models to predict the Mean Opinion Score (MOS) for proactive network management, improving service quality monitoring.
  • Real-Time Monitoring: Integrated MOS predictions into operational dashboards, enabling real-time tracking of network performance and early degradation alerts.
  • Improved Accuracy: Achieved 94% accuracy in MOS predictions using advanced regression models like Random Forest, XGBoost, and LSTM.
  • Proactive Alerts: Enabled early detection of potential network issues, reducing customer complaints by 22% and improving SLA adherence..

Technologies and Tools

  • Programming Languages: Python
  • Machine Learning Libraries: TensorFlow, Scikit-learn
  • Data Handling: Pandas, NumPy
  • Databases: SQL for data storage and retrieval
  • Containers: Docker for system deployment and scalability
  • Development Tools: Jupyter Notebooks for exploratory data analysis and model development

Problem Statement

Traditional inventory management systems often fail to predict stock levels accurately, leading to frequent stockouts. This disrupts supply chains, increases operational costs, and negatively impacts customer satisfaction. The C5i project aimed to solve this problem by developing a predictive model to optimize inventory management.

Solution Approach

Collected and preprocessed historical inventory data, sales data, and external factors like weather and promotions. Built a time-series forecasting model using TensorFlow to predict future stock levels. Designed a system to integrate real-time data from various sources (weather, promotions, and demand forecasts) to continuously adjust predictions. The system was deployed using Docker for easy scalability and integration with existing inventory management systems.

Results

  • 30% improvement in forecasting accuracy, leading to better stock level predictions.
  • 25% reduction in stockouts, which increased operational efficiency and reduced inventory costs.
  • Real-time adjustments allowed for better decision-making, reducing overstocking and stockouts.

Challenges

Handling missing or noisy data required the implementation of advanced data preprocessing techniques. Integrating multiple data streams in real time required robust pipeline architecture, solved using Docker for deployment.

Future Work

Expand the model to include real-time inventory tracking for even more accurate predictions. Integrate with ERP and supply chain management systems for seamless automation and decision-making.
