Agent Attrition Prediction Model

Project Overview

This project predicts future agent performance (Months 4–6) using insights from the first three months of activity. By analyzing early sales momentum, we classify agents into High Performing (AC ≥ 15,000) or Others, enabling proactive coaching, better onboarding support, and improved productivity outcomes.

Key Features

  • Cohort-Based Structure: Agents grouped quarterly based on tenure for fair performance comparison.
  • Early Signal Modeling: Uses AC, AFYP, policy counts, and proposal activity from the first 3 months.
  • Multi-Model Benchmarking: Evaluated tree-based models and neural networks to identify the champion model.
  • Time-Based Validation: Separate unseen cohorts used to validate real-world performance.
  • Explainable Insights: Feature importance analysis to highlight early performance drivers.
  • Deployment Ready: Outputs integrated into dashboards for managerial decision-making.

Technologies and Tools

  • Programming: Python
  • Modeling: Random Forest, XGBoost, LightGBM, CatBoost, NeuralNetFastAI
  • Feature Selection: Statistical analysis & ML-based importance methods
  • Data Processing: Pandas, NumPy
  • Visualization: Power BI
  • Storage: AWS S3 data pipelines

Problem Statement

Insurance organizations often evaluate performance too late, limiting opportunities for early intervention. Without predictive signals, managers cannot prioritize coaching or identify high-potential agents quickly. The goal was to create a data-driven early forecasting system that improves productivity and retention through timely decision support.

Solution Approach

We structured the data into tenure-based cohorts and engineered performance features from the first three months. Multiple classification models were trained and evaluated using time-based splits to ensure realistic deployment conditions.

  • Data Preparation: Filtering, cleaning, and cohort alignment.
  • Feature Engineering: Monthly AC, AFYP, policies, and proposals.
  • Model Training: Supervised classification with ROC-AUC evaluation.
  • Champion Selection: Selected model balancing accuracy, recall, and stability.
  • Deployment: Probability scores surfaced for managerial action.

Results

  • Stable Validation Performance: Maintained strong ROC-AUC across unseen cohorts.
  • Balanced Precision & Recall: Effective identification of high-performing agents.
  • Improved Early Differentiation: Clear separation between high-potential and low-momentum agents.

Note: Final metrics may vary depending on cohort mix and deployment conditions.

Challenges

Managed class imbalance, prevented data leakage across cohorts, ensured stable time-based validation, and balanced model complexity with interpretability for leadership stakeholders.

Future Enhancements

  • Behavioral & Engagement Signals: Include manager and branch-level indicators.
  • Cost-Sensitive Thresholding: Optimize predictions based on revenue impact.
  • Automated Scoring Pipeline: Real-time evaluation for new agents.
  • Workflow Integration: Connect outputs directly with coaching and performance systems.
